这是一个c/c++的项目，该项目部署在3588开发板上，运行环境是ubuntu22.04.该项目的主要功能如下：
1. 注册到mqtt，接收mqtt下发的分析命令，主要包括
  1.1 检测场景（启用的功能ID）；
  1.2 检测区域（4元组标识）；
  1.3. 需要过滤的区域（4元组标识）；
  1.4. 检测阈值（置信度）；
  1.5. 检测帧率（每秒多少帧）；
  1.6. 功能激活码；
  1.7. 其他(json)
  
2. 根据不同的场景加载不同的模型，模型包括以下几种：
  2.1 CNN的图片识别模型
  2.2 基于yolo11的图片识别模型
  每个模型都有自己的编号以适应不同的场景启用。
  
3. 本地有一项配置，包括：
  3.1 远端的mqtt服务器地址
  3.2 远端的mqtt订阅的频道，以及接收服务段数据的处理功能
  3.3 分析完结果提交给mqtt当前的检测结果
  3.4 rtsp流拉取协议的ip和port等信息
  3.5 本地服务名称
  
任务处理流程如下：
1. 启用本服务，服务启动后，读取本地配置文件：local.config.json，获取对应配置
2. 注册到远端的mqtt服务，监听远端服务推送的消息，包括本地ip，名称，等信息
3. 接收到mqtt下发的分析命令，根据不同的场景加载指定的模型
4. 通过rtsp协议拉取远端的视频流，根据mqtt下发的频率参数，每隔几秒获取一张图片
5. 将图片丢入对应的模型分析，同时记录该图片对应视频的时间戳
6. 获取结果，并组织对应大结果信息
7. 将最终的结果推送到mqtt服务，结果包括：(1). 类别。(2). 识别的图片的目标区域。(3). 当前所属视频的时间戳
